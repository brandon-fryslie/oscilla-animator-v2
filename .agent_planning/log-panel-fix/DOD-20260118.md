# Log Panel Fix - Definition of Done

**Date**: 2026-01-18
**Topic**: log-panel-fix

## Acceptance Criteria

### Functional Requirements

- [ ] LogPanel displays in the bottom area as a tab
- [ ] LogPanel is next to the Diagnostics panel (both tabs in same group)
- [ ] Users can switch between Console and Logs tabs
- [ ] LogPanel shows logs from `rootStore.diagnostics.logs`
- [ ] Logs display with proper color coding (error=red, warn=yellow, info=blue)
- [ ] Log panel auto-scrolls when new logs arrive
- [ ] Panel is resizable (height can be adjusted)

### Technical Requirements

- [ ] Dockview panel wrapper created at `src/ui/dockview/panels/LogPanel.tsx`
- [ ] LogPanel registered in `PANEL_DEFINITIONS` array
- [ ] LogPanel added to `PANEL_COMPONENTS` object
- [ ] LogPanel added to `'bottom-left'` group in default layout
- [ ] Tab order: Console first, Logs second
- [ ] Follows existing panel wrapper pattern (implements `IDockviewPanelProps`)

### Code Quality

- [ ] TypeScript compiles without errors
- [ ] No new warnings in console
- [ ] Follows existing code patterns and conventions
- [ ] Proper imports and exports

### Verification

- [ ] Application starts without errors
- [ ] Bottom panel shows two tabs: "Console" and "Logs"
- [ ] Clicking "Logs" tab displays the log panel
- [ ] Logs are visible if any exist in the store
- [ ] Auto-scroll works when new logs are added
- [ ] Can switch between Console and Logs tabs smoothly
- [ ] Panel height is adjustable via Dockview separator

## Out of Scope

- Changes to LogPanel component itself (already functional)
- Changes to DiagnosticConsole panel
- Changes to DiagnosticsStore
- Adding new features to logging system
- Filtering or searching logs (future enhancement)
- Log export functionality (future enhancement)
