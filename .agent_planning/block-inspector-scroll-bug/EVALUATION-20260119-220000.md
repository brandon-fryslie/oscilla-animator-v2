## Problem Statement

The Block Inspector panel does not scroll when content exceeds the viewport height. Users cannot see/edit blocks with many inputs, outputs, or parameters.

---

## Root Cause Analysis

### Comparison: BlockLibrary (works) vs BlockInspector (broken)

**BlockLibrary** (`src/ui/components/BlockLibrary.tsx` + `.css`):
```css
.block-library {
  height: 100%;
  display: flex;
  flex-direction: column;
  overflow: hidden;
}

.block-library__categories {
  flex: 1;
  overflow-y: auto;
  min-height: 0;  /* Critical for flex child scrolling */
}
```

**BlockInspector** (`src/ui/components/BlockInspector.tsx`):
```tsx
// No container element - renders content directly with inline styles
<div style={{ padding: '16px' }}>
  {/* content */}
</div>
```

### The Issue

BlockInspector has no scroll container. Content grows beyond viewport with no scroll mechanism.

Key CSS requirements for flex child scrolling:
1. Container: `height: 100%` + `display: flex` + `flex-direction: column` + `overflow: hidden`
2. Scroll area: `flex: 1` + `overflow-y: auto` + `min-height: 0`

---

## Solution

Add a wrapper container with proper flexbox layout to BlockInspector, matching BlockLibrary's pattern.

**Option A: CSS file (preferred)**
- Create `BlockInspector.css` with identical pattern to BlockLibrary
- Import CSS in component
- Use CSS classes instead of inline styles

**Option B: Inline styles**
- Add wrapper div with inline styles matching the pattern
- Less maintainable but no new file

**Recommendation**: Option A - follows existing codebase pattern (BlockLibrary has CSS file)
